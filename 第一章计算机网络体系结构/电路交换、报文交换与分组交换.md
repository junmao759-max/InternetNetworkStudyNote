###### 电路交换、报文交换与分组交换
###### - 电路交换：
- 最典型的电路交换网是传统电话网，交换指的是以某种方式==动态==的分配传输线路的资源。[^1]
###### - 优点
1. 通信时延小：因为通信线路为通信双方专用，数据直达，所以传输效率高。
2. 有序传输：双方通信时按发送顺序传送数据。
3. 没有冲突：不同的通信双方有不同的信道，不会出现争用信道的问题。
4. 实时性强：通信双方之间的物理通路一旦建立，双方就可随时通信。
###### - 缺点
 1. 建立连接时间长：电路交换的平均时间对计算机通信来说太长。
 2. 线路利用率低：物理通路被通信双方独占，即使线路空闲，也不能供其他用户使用。
 3. 灵活性差
 4. 难以实现差错控制

###### - 报文交换：
- 数据交换的单位是报文，报文交换采用存储转发技术，整个报文先传送到相邻节点，全部存储之后查找转发表，转发到下一个节点，如此重复，直至达到目的节点。
###### - 优点
1. 无建立连接时延：用户可随时发布报文
2. 灵活分配线路
3. 线路利用率高
4. 支持差错控制
###### 缺点
1. 转发时延高
2. 缓存开销大
3. 错误处理低效

###### 分组交换：
也是基于存储转发技术，但是对于长报文可以分成若干个较小的等长数据段（除了最后一个数据段），在每个数据段前面添加一些有必要控制信息组成的首部，构成分组
###### 优点

1. 方便存储管理，存储转发开销小。分组长度固定，对应缓冲区大小固定。
2. 传输效率高。分组逐个传输，后一个分组存储操作与前一个分组转发操作并行，流水线方式减少报文整体传输时间。
3. 减少出错概率与重传代价。分组数据量短，出错概率低，重传数据量大幅减少，提升可靠性、降低传输时延。

###### 缺点

1. 存在存储转发时延。相比电路交换传输时延更大，交换机转发分组时需要缓存，且节点需要更强处理性能。
2. 传输额外信息量增加。每个小段分组都要附加头部控制信息，总传输数据量增大 5%~10%，控制逻辑复杂，降低通信效率。
3. 数据报服务易失序。采用无连接数据报服务时，分组到达目标主机路径不同，会出现失序、丢包、重复分组问题；需要额外对分组编号、重排序，处理繁琐。

 > 补充：虚电路服务无失序问题，但需要呼叫建立、数据传输、虚电路释放三步流程；连续大批量数据传输场景下，传输耗时高于电路交换。
 

[^1]: 电路交换分三步：建立连接、传输数据、释放连接。
