###### 1. 速率（Speed / 数据传输速率）

1. 定义：主机在数字信道上传送数据的速度，也称**比特率**。
2. 单位：b/s（比特 / 秒，bps）；高数据速率用 kb/s、Mb/s、Gb/s。
3. 说明：描述**数字信道理论最大传输速率**。

###### 2. 带宽（Bandwidth）

1. 物理原意：信号允许通过的**频率范围**，单位赫兹（Hz）。
2. 计算机网络含义：信道所能传输的**最高数据传输速率**，单位 b/s，是速率的同义描述。

###### 3. 吞吐量（Throughput）

1. 定义：单位时间内，通过网络 / 信道 / 接口**实际的数据量**。
2. 用途：衡量网络真实负载，反映实际传输能力，数值通常低于带宽上限。

###### 4. 时延（Delay / 延迟）

总时延 = **发送时延 + 传播时延 + 处理时延 + 排队时延**

###### （1）发送时延（传输时延）

公式：发送时延=发送速率(b/s)分组长度(bit)​ 含义：将分组所有比特推向链路所花费的时间，和分组长度、发送速率相关。

###### （2）传播时延

公式：传播时延=电磁波在介质上的传播速率(m/s)信道长度(m)​ 含义：单个比特在物理链路上从一端传到另一端的时间，只和距离、介质有关，与分组大小无关。

###### （3）处理时延

含义：交换机 / 路由器收到分组后，解析首部、查路由、差错校验花费的时间。

###### （4）排队时延

含义：分组到达交换机时，若链路繁忙，在输入 / 输出队列排队等待转发的时间，网络拥堵时会大幅增大。

###### 5. 往返时间 RTT（Round-Trip Time）

1. 定义：发送端发出分组，到收到对方确认应答的总耗时。
2. 包含：发送时延、传播时延、中间节点处理 / 排队时延、应答分组的全部时延。

###### 6. 信道利用率

1. 公式：信道利用率=总时间有数据通过的时间​
2. 含义：信道被数据占用的时间占比。
3. 特点：
    - 利用率过低：带宽资源浪费；
    - 利用率过高：排队时延急剧增加，极易网络拥塞。

###### 补充易混区分

1. 发送时延：和**分组长度、带宽**相关；
2. 传播时延：只和**物理距离、传输介质**相关；
3. 带宽比喻：链路如同空心管道，带宽 = 管道横截面积，传播时延 = 管道长度。
